# Antigravity 행동 규칙 (Rules)

## 코드 직접 수정 제한
- 파일을 직접 수정하는 도구(write_to_file, replace_file_content, multi_replace_file_content 등)를 사용하기 전에, 반드시 사용자에게 수정 계획을 먼저 설명하고 승인을 받아야 합니다.
- 사용자가 "코드를 수정해 줘", "적용해 줘" 등 명시적인 지시를 내리기 전에는 임의로 프로젝트 파일을 변경할 수 없습니다.
- 항상 수정할 내용을 코드 스니펫(Diff 또는 전체 코드)으로 대화창에 먼저 제안하고 의견을 구하세요.
